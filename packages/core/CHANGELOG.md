# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2025-01-14

### Added
- Initial release of @healthsync/core
- Universal data model for health data integration
- Plugin system for extensible platform support
- Type-safe TypeScript interfaces
- Configuration management
- Data transformation utilities
- Comprehensive test coverage
- Full TypeScript support with type definitions
- ESM and CommonJS module support
- Zero external dependencies for core package

### Features
- **Unified Data Model**: Standardized health data format across all platforms
- **Plugin Architecture**: Extensible system for adding new health data sources
- **Type Safety**: Full TypeScript support with comprehensive type definitions
- **Multi-Module**: ESM and CommonJS exports for maximum compatibility
- **Lightweight**: Zero dependencies, minimal bundle size
- **Well-Tested**: Comprehensive test coverage with Jest

### Documentation
- Complete API documentation
- Integration guides for common use cases
- Plugin development guide
- Migration guides from other health SDKs
